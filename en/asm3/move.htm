If the level of nested subassemblies exceeds 1, mover don't work as expected when it comes to moving parts. Instead of moving just the part, the whole assembly moves. What is going on here…?

Creating the Repository

  1. Switch to the Part Design workbench.
  2. Create a new file, and save it under the work file name "move.fcstd".
  3. Create a new Group node and name it "Repository".

We will work with a single component, a sphere:

  1. Select the document node in the Model Tree.
  2. Create a new Part Design > Body.
  3. Create a new Part Design > Sketch on the XY plane.
  4. Create a Sketch > Geometries > Arc by center.
  5. Constrain the arc center on the sketch origin.
  6. Constrain the end points on the X axis.
  7. Constrain the radius to "5 mm".
  8. Create a Sketch > Geometries > Line.
  9. Constrain the end points of the line to the end points of the arc.

The sketch is fully constrained now:

  1. Leave the sketch.
  2. Create a new Part Design > Revolution.
  3. Select the horizontal sketch axis and click OK.
  4. Collapse the Body nodes.

Now wrap the body with an Assembly:

  1. Switch to the Assembly 3 workbench.
  2. Create a new Assembly.
  3. Set the Label property to "Sphere".
  4. Drag the Body on the Sphere node.
  5. Collapse the child nodes.
  6. Move the Sphere to the Repository.
  7. Hide the Sphere.
  1. Save a copy of the project file under the name "move-001.fcstd".

The components have been created, now we will create the nested assembly structures.

Moving Components

In this section we will move components in space.

You have already seen the Placement > Position property work as expected. Now we will work the the Axial Mover, Assembly Part Mover and the Quick Mover tools.

If you want to quick-start from here, load the file move-003.fcstd, and save it under the work file name "move.fcstd".

Moving Sphere 1

We will start with the Axial Mover on Sphere 1.

  1. Press 1 on the keyboard to switch to the front view.
  2. Select Sphere 1 in the Model Tree. This object is a Part of the Project Assembly Container.
  3. Click the menu Assembly3 > Axial move part.
    A tool appears in the 3D window that looks and works exactly like the standard Freecad Transform tool.
  4. Drag the blue arrow to the side, then press Esc.
    The topmost sphere moved as expectd.
  5. Undo the move command.

Now we will try the Assembly Part Mover

  1. Press 0 to get to the standard isometric view.
  2. Select Sphere 1 in the Model Tree.
  3. Click the menu Assembly3 > Axial move part.
    A tool with a set of three perpendicular rings and arrows appears in the 3D window.
  4. Press 1 to show the standard front view.
  5. Drag the mover arrows in the center to the side, then
    The topmost sphere moved as expectd.
  6. Note that the Project Assembly Container is highlighted in the Model Tree.
    This indicates that the node is being edited.
  7. Press Esc.
  8. Undo the move command.

If you drag an arrow cross of the Assembly Part Mover, you can move the part in the plane of the arrow cross. If you drag a ring, the part is rotated about the ring axis (which is not easily seen on a sphere…).

Now we will try the Quick Mover.

  1. Select Sphere 1 in the Model Tree.
  2. Click the menu Assembly 3 > Quick move.
  3. Move the mouse pointer. The selected object follows the pointer until you click.
  4. Note that the parent node of the moved object is selected, Parts001.
  5. Undo the move command.

Three different tools to move objects. Choose the one that suits you preferences best.

Conclusion

Although the mover tools seem to behave "erratic" when subassemblies or components in subassemblies are moved, but there is a method to the madness:

  • Components in the highest level (zero) assembly are moved as usual.
  • Components in subassemblies always move the Level 2 Assembly.

This does not make much sense if you select nodes in the Model Tree to move them, but it does make perfect sense if you select components in the 3D window: Because assembly containers have no visual representation in the 3D window, it cannot be decided when you select Sphere 4 whether you want to move the sphere, or its parent, Level 4 Assembly, or its parent Level 3 Assembly, or its parent Level 2 Assembly. The only certainty is that the first level assembly, the Project Assembly Container, can never be moved, because this element provides the project-wide global coordinate system.

The Double Selection Trick

But what if you really wanted to move a component further down the hierarchy? There are two options:

  • Drag the parent assembly container on the project document node, move your components, and drag the parent assembly container back to where it belongs.
  • Use the double-selection trick.

The double selection trick is simple: Select the object to be moved, and its parent container. But before checking that out, let's add another sphere to the Level 4 Assembly:

  1. Right-click the Sphere in the Repository, then click Link actions > Make link.
  2. Rename it to Sphere 4b.
  3. Drag it on Level 4 Assembly.
  4. Switch to the View Tab on the Property Panel.
  5. Set Override Material to "true".
  6. Select a color of your choice into the Shape Material property.
  7. Switch to the Data Tab.
  8. Set the Placement > Position > x property to "15 mm".
  9. Set the z property to "-45 mm".

The new sphere should appear at the side of Sphere 4. Now Let's test it.

  1. Select Sphere 4 and Level 4 Assembly.
  2. Move the sphere somewhat to the right.

That works, Sphere 4 moved.

  1. Undo the move command.
  2. Select Sphere 4 and Level 3 Assembly.
  3. Select the mover tool of your choice from the Assembly3 menu.
  4. Move the sphere somewhat to the right.

Level 4 Assembly moved.

  1. Undo the move command.
  2. Select Sphere 4 and Level 2 Assembly.
  3. Select the mover tool of your choice from the Assembly3 menu.
  4. Move the sphere somewhat to the right.

Level 3 Assembly moved.

  1. Undo the move command.
  2. Select Sphere 4 and Project Assembly Container, which is the first level assembly.
  3. Move the sphere somewhat to the right.

Level 2 Assembly moved.

Conclusion

The examples in the previous section showed that there is a set of simple rules, followd by the movers:

1. The selected component low in the hierarchy is used as a grip.

2. The Assembly below the higher selected component is moved.

3. If only one component was selected, the highest assembly is considered to be selected implicitly.
